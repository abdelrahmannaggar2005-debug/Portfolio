The Challenge

GaitAI needed a reliable mobile front-end that could capture user data, sync it instantly with the cloud, and communicate seamlessly with an AI-driven backend for gait analysis — all while keeping the experience smooth and consistent across Android and iOS.

My Approach

  1. App Foundation: Built the Flutter application from the ground up, structuring screens and state management to support the project's analysis workflow.
  2. Cloud Integration: Connected the app to Firebase Auth, Firestore, and Firebase Storage to handle authentication, data persistence, and media uploads in real time.
  3. Backend Bridge: Developed a Flask REST API in Python to link the mobile app with the AI processing pipeline, ensuring smooth, reliable data exchange.

The Impact

The result was a fully functional cross-platform application with a stable backend bridge — giving the GaitAI system a reliable way to collect, store, and exchange data between users and the AI analysis engine.

What I Learned

  • ▸
    Connecting a mobile front-end to multiple backend services (Firebase + custom API) requires careful handling of data consistency and error states.
  • ▸
    Designing REST APIs with Flask taught me how to structure clean, predictable endpoints for cross-team collaboration with AI/ML components.
  • ▸
    Cross-platform development with Flutter significantly speeds up delivery without compromising native-like performance.
